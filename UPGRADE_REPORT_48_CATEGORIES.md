### 2. 评分标准设计 ✅

#### 设计原则
根据不同类目的行业特性，调整5个维度（ROI、退货率、利润率、店铺评分、运营时长）的阈值：

**高毛利类目**（珠宝、美容仪器、保健品等）
- ROI要求较高（≥2.5%）
- 退货率要求严格（≤15%）
- 利润率要求高（≥30%）
- 运营门槛高（≥12月）

**快消品类目**（食品、日用百货、个人护理等）
- ROI适中（≥1.5%）
- 退货率宽松（≤20%）
- 利润率适中（≥12%）
- 运营门槛低（≥6月）

**标品类目**（家电、数码、图书等）
- ROI偏低（≥1.2%）
- 退货率严格（≤10%）
- 利润率偏低（≥8%）
- 店铺评分要求高（≥4.0）

**服饰类目**（女装、男装、童装等）
- ROI适中（≥1.8%）
- 退货率较高（≤35%）
- 利润率适中（≥15%）
- 季节性强，运营≥9月

**特殊监管类目**（保健食品、滋补品等）
- ROI较高（≥1.8%）
- 退货率极低（≤8%）
- 利润率高（≥30%）
- 店铺评分≥4.0，运营≥9月

#### 评分权重（总分100）
- ROI（投资回报率）：25分
- 退货率：25分
- 净利润率：25分
- 店铺评分：12.5分
- 运营时长：12.5分

## 📁 技术实现

### 数据库迁移
**文件**：`migrations/0005_douyin_48_categories.sql`

**操作**：
1. 清除旧31个品类数据
2. 插入新48个品类配置（每类5个字段，共240条记录）
3. 每个字段包含：
   - 字段名、标签、类型
   - 阈值、比较运算符
   - 最高分、评分规则JSON
   - 是否必填、显示顺序

**示例**（女装品类）：

### 前端实现
**文件**：`public/static/app-extended.js`

**核心功能**：
```javascript
// 1. 搜索过滤函数
function filterCategories(searchText) {
  const search = searchText.toLowerCase().trim();
  const filtered = allCategories.filter(cat => {
    const catLower = cat.toLowerCase();
    const pinyin = pinyinMap[cat] || '';
    return catLower.includes(search) || pinyin.includes(search);
  });
  renderCategoryTabs(filtered);
}

// 2. 标签渲染函数
function renderCategoryTabs(categories) {
  tabsContainer.innerHTML = categories.map(cat => `
    <button class="px-4 py-2 rounded transition ${isActive}">
      ${cat}
    </button>
  `).join('');
}
```

**HTML结构**：
```html
<!-- 搜索框 -->
<input id="category-search" placeholder="搜索品类（支持拼音首字母）..."
       oninput="filterCategories(this.value)">

<!-- 标签容器 -->
<div id="category-tabs" class="flex flex-wrap gap-2">
  <!-- 动态加载品类标签 -->
</div>
```

## 🎯 使用指南

### 管理员
1. 登录后台：https://webapp.pages.dev/admin/login
2. 点击"评分配置" → 进入配置管理页面
3. 使用搜索框快速定位品类（支持中文/拼音）
4. 点击品类标签查看/编辑配置
5. 修改阈值、权重后点击"保存配置"

### 融资方用户
1. 注册/登录系统
2. 填写项目信息时，在品类下拉框中：
   - 直接输入品类名称（如"女装"）
   - 或输入拼音首字母（如"nz"）
3. 系统自动过滤并显示匹配的品类
4. 选择品类后，系统会自动应用该品类的评分标准
